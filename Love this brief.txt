Why this matches the Apple docs aesthetic

Typography (SF on Apple devices, system stack elsewhere).
We use the system font stack (-apple-system, system-ui, etc.), so San Francisco renders on Apple devices without bundling fonts—matching HIG guidance to use system fonts for legibility and dynamic behavior; custom fonts should match system behaviors if used.

Color (semantic + light/dark).
Tokens follow Apple’s approach: semantic system colors that adapt to light/dark rather than hard‑coding hex values. In CSS we provide approximations (--blue, --indigo, etc.) but keep them semantic so you can tune later.

Materials (blur, translucency, hairlines).
Navbar and some surfaces use translucent backgrounds with backdrop blur and hairline borders to create depth and context separation—exactly what Apple’s Materials guidance describes.

Icons (symbol tiles).
We include symbol‑tile containers that look like SF Symbols tiles, but we don’t embed SF Symbols because Apple’s license restricts them to Apple‑platform apps (not general websites). You can drop in your own SVGs or use an open set (e.g., Lucide) styled to the tile.

Key design tokens (edit in src/styles/theme.css)

:root {
  --content-max: 1100px;
  --radius: 14px;                    /* legacy card corners */
  --fs-hero: clamp(44px,3vw+28px,68px);
  --fs-h1: clamp(34px,2.4vw+20px,48px);
  --fs-h2: clamp(26px,1.6vw+18px,34px);

  /* dynamic-ish palette; keep semantic */
  --blue:#0a84ff; --green:#34c759; --indigo:#5e5ce6;
  --teal:#64d2ff; --orange:#ff9f0a; --pink:#ff375f;

  /* dark defaults (light tokens included too) */
  --bg:#111214; --surface:#1a1b1e; --subtle:#0f1012;
  --hairline:#2a2b2e; --text:#e7e7ea; --muted:#a7aab0;
}

	•	Cards: .card = surface + 1px hairline + hover lift
	•	Symbol tiles: .symbol.symbol--blue|indigo|teal = rounded tiles behind mini icons
	•	Hero: .badge glass pill + “dot” accent
	•	Sidebar: sticky, independently scrollable; filter with / key; mobile drawer

Friendly legal & design notes
	•	Fonts: Don’t self‑host San Francisco on the web; use the system stack to get SF on Apple OSs and similar fonts elsewhere. Apple’s guidance emphasizes system fonts and dynamic behaviors.
	•	SF Symbols: Licensed for Apple platforms; not for general web use—so keep the stylistic tiles and use non‑Apple SVGs you own.
